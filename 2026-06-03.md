## Exercism notes

### acronym
- Solution:
```python
def abbreviate(words):
    return ''.join(word[0].upper() for word in words.replace('-', ' ').replace('_', ' ').split())
```
- Pass test: có
- Học được: `''.join()` + generator expression; `word[0]` an toàn vì `split()` không bao giờ trả chuỗi rỗng
- Note cho sau: nếu separator nhiều hơn 2 ký tự → dùng `re.sub(r'[-_]', ' ', words)` thay chuỗi `.replace()`. Với đúng 2 ký tự thì bản hiện tại dễ đọc hơn.

### pangram
- Solution (bản đã nộp):
```python
def is_pangram(sentence):
    for char in 'abcdefghijklmnopqrstuvwxyz':
        if char not in sentence.lower():
            return False
    return True
```
- Pass test: có
- Bản Pythonic hơn (áp set idiom đã tự ghi nhận ở isogram Day 8):
```python
import string

def is_pangram(sentence):
    return set(string.ascii_lowercase) <= set(sentence.lower())
```
- Học được: `<=` là "tập con" (alphabet ⊆ ký tự của câu); O(n) thay vì O(26n); dùng `string.ascii_lowercase` thay vì hard-code 26 chữ cái.

## Jinja2 config-gen (món chính)

**Insight:** đây là hiện thân kỹ thuật của intent-vs-config (gap tự chỉ ra ở Day 1). YAML = intent, template = cách render, output = config. Lần đầu chạm tư duy intent-based bằng code mình kiểm soát được, và nhìn output IOS là biết đúng/sai ngay, không cần lab.

### Bản đầu (tự viết) — chạy được, deploy được, nhưng indent lệch
- Logic đúng: `for ... if vlan.ip is defined` loop-filter, nested `{% if vlan.description is defined %}`, `yaml.safe_load`, `encoding="utf-8"` cho cả đọc lẫn ghi.
- Vấn đề: indent block SVI không đều — `description`/`ip address` ra 2 space, `no shutdown` ra 1 space.
- Nguyên nhân: điều khiển whitespace bằng tay với `-%}`, cộng leading space đứng trước `{% if %}`/`{% endif %}` rò ra output rồi cộng dồn với space của dòng nội dung.

### Bản fix — dùng Environment + 2 cờ, vứt hết `-%}`
```python
#!/usr/bin/env python3
"""Sinh config Cisco IOS (VLAN + SVI) tu file YAML."""
from pathlib import Path
import yaml
from jinja2 import Environment

TEMPLATE = """\
! --- VLAN Configuration ---
{% for vlan in vlans %}
vlan {{ vlan.id }}
 name {{ vlan.name }}
{% endfor %}
! --- SVI Configuration ---
{% for vlan in vlans if vlan.ip is defined %}
interface Vlan{{ vlan.id }}
    {% if vlan.description is defined %}
 description {{ vlan.description }}
    {% endif %}
 ip address {{ vlan.ip }} {{ vlan.mask }}
 no shutdown
!
{% endfor %}"""

ENV = Environment(trim_blocks=True, lstrip_blocks=True, keep_trailing_newline=True)


def load_vlans(path: Path) -> list[dict]:
    """Doc YAML, tra ve list cac dict vlan."""
    data = yaml.safe_load(path.read_text(encoding="utf-8"))
    return data["vlans"]


def render_config(vlans: list[dict]) -> str:
    """Render config IOS tu list vlan."""
    return ENV.from_string(TEMPLATE).render(vlans=vlans)


def main() -> int:
    vlans = load_vlans(Path("vlans.yaml"))
    config = render_config(vlans)
    print(config)
    Path("vlan_config.txt").write_text(config, encoding="utf-8")
    return 0


if __name__ == "__main__":
    import sys
    sys.exit(main())
```

### Hai cờ làm gì
- `trim_blocks=True`: xoá newline ngay sau `%}` → tag `{% for %}` đứng riêng một dòng không đẻ ra dòng trống, không cần `-%}` nữa. (Đây là cờ Ansible bật mặc định, nên học bây giờ là front-load cho tuần 22.)
- `lstrip_blocks=True`: xoá whitespace từ đầu dòng đến tag, **chỉ** với dòng chỉ chứa block tag → indent `{% if %}`/`{% endif %}` cho dễ đọc (thụt 4 space) mà output không dính space đó. Dòng nội dung như ` description` thì giữ nguyên 1 space.

Output giờ đều 1 space, sạch:
```
interface Vlan10
 description Quan ly thiet bi
 ip address 10.0.10.1 255.255.255.0
 no shutdown
!
```

### Bài học
- Đừng điều khiển whitespace bằng tay từng chỗ → không scale khi template nhiều nhánh.
- Config sạch quan trọng cho **diff** giữa 2 version + **idempotency** sau này, không chỉ "nạp được lên switch".
- config-gen = chiều **push**; device-check = chiều **pull**. Tuần 3 SSH thật → ráp thành vòng tròn automation hoàn chỉnh.
