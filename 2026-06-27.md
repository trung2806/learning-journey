## Việc làm hôm nay: migrate main_loop() sang asyncio + fan-out đa node với asyncssh

`main_loop()` từ D37 là hàm đồng bộ poll một node duy nhất. Hôm nay refactor toàn bộ Imperative Shell sang async, thêm transport layer asyncssh, implement fan-out N node song song với per-node state isolation.

**Chuỗi quyết định (mentor dialogue, Trung tự chốt từng bước):**

1. **threading vs asyncio — chọn asyncio.** Cooperative multitasking: thấp RAM (KB/coroutine vs MB/thread), không cần Lock ở tầng ứng dụng, `task.cancel()` graceful shutdown. threading phù hợp blocking I/O ở quy mô nhỏ; asyncio phù hợp I/O-bound network polling nhiều node.

2. **asyncssh, không phải netmiko + run_in_executor.** netmiko là blocking I/O — bọc trong executor chỉ là threading giấu mặt, không có lợi ích asyncio. asyncssh là native async SSH. JSON-RPC (httpx) tối ưu hơn cho SR Linux thế hệ mới, nhưng SSH vẫn chiếm đa số thiết bị thực tế — chọn asyncssh cho mục tiêu học thuật.

3. **`async def main_loop()` + `asyncio.run()` entrypoint.** Gọi coroutine không có `asyncio.run()` chỉ trả về coroutine object, không chạy gì. `asyncio.run()` là cầu nối đồng bộ duy nhất còn sót lại: khởi tạo event loop, chạy, dọn dẹp.

4. **`asyncio.Event()` + `loop.add_signal_handler()`.** `threading.Event` không an toàn khi set từ asyncio context. `loop.add_signal_handler()` schedule callback an toàn bên trong event loop — Unix-only, phải gọi từ running loop, main thread only (asyncio.run() đảm bảo).

5. **`asyncio.wait_for(_stop_event.wait(), timeout=interval)` — tighter sleep.** `await asyncio.sleep(T)` ngủ cứng hết T giây sau khi SIGTERM. `wait_for` tỉnh dậy ngay khi stop_event set, dù chưa hết timeout.

6. **`asyncio.TaskGroup` cho fan-out.** Structured concurrency: khi thoát `async with`, toàn bộ tasks đã done — không có orphaned task. Nếu task fail, TaskGroup cancel toàn bộ còn lại + raise `ExceptionGroup`. So với `gather()`: gather để tasks chạy orphan khi có exception.

7. **Option A — N tick() calls, không aggregate.** `tick()` không biết fleet topology, chỉ nhận raw_data của một node. Imperative Shell giữ `state_registry: Dict[str, SystemAlertState]` — fault containment hoàn toàn, Node A chết không ảnh hưởng Node B. Cooldown của từng node độc lập.

8. **`poll_node` là error boundary — swallow + fallback.** Trả về dict với `healthz.status = "unreachable"` khi lỗi SSH. Hệ quả: `task.result()` không bao giờ raise, `except*` trong TaskGroup là dead code. Hai lớp xử lý lỗi không nhất quán — chấp nhận cho lab, production phải chọn một.

## Ruling

- **`asyncio.run()` là cầu nối đồng bộ duy nhất.** Gọi coroutine mà không có nó = tạo object nằm im, không chạy. Nó khởi tạo, chạy, dọn event loop.
- **`loop.add_signal_handler()`, không phải `signal.signal()` trong asyncio.** Signal handler gọi asyncio primitive trực tiếp có thể race với loop. `add_signal_handler()` schedule callback an toàn, Unix-only.
- **`asyncio.wait_for(event.wait(), timeout=T)` cho interruptible sleep.** Tỉnh dậy khi event set HOẶC khi timeout — không block cứng như `sleep(T)`. Pattern chuẩn cho monitoring loop responsive.
- **`TaskGroup` = no orphaned task.** Scope boundary tường minh: khi thoát `async with`, mọi task đều done. `ExceptionGroup` thay vì Exception đơn — bắt bằng `except*`.
- **`except BaseException` cho `CancelledError`.** Nếu task bị cancel, `task.result()` raise `CancelledError` — là `BaseException`, không phải `Exception`. `except Exception` không bắt được.
- **Error boundary phải nhất quán.** Hoặc transport swallow (poll_node trả fallback, TaskGroup không cần except*), hoặc transport raise (TaskGroup xử lý). Không cả hai cùng lúc.

## Glossary mới

- **`asyncio.TaskGroup` (Python 3.11+).** `async with asyncio.TaskGroup() as tg:` — context manager tạo và quản lý nhóm task. Khi thoát block: tất cả task đã done. Nếu task fail: cancel toàn bộ còn lại, raise `ExceptionGroup`. Structured concurrency — không có task rò rỉ ra ngoài scope.
- **`ExceptionGroup` + `except*`.** Khi nhiều task fail đồng thời trong TaskGroup, Python bọc tất cả vào `ExceptionGroup`. Bắt bằng `except* SomeError as eg:` (dấu `*` bắt buộc). `except Exception` không bắt được `ExceptionGroup`.
- **`asyncio.wait_for(coro_or_future, timeout)`.** Chạy coroutine với deadline. Hết timeout → raise `asyncio.TimeoutError`. Kết hợp với `stop_event.wait()` tạo interruptible sleep.
- **`loop.add_signal_handler(signum, callback)`.** Đăng ký signal handler an toàn với running event loop — Unix-only. Callback chạy trong loop thread, không race với asyncio primitives. Gọi qua `asyncio.get_running_loop()`.
- **`known_hosts=None` (asyncssh).** Bỏ SSH host key verification — tương đương `StrictHostKeyChecking=no`. Lab only; production cần `known_hosts` hoặc CA.
- **TEST-NET-1 (`192.0.2.0/24`).** RFC 5737 — dải IP dành riêng cho documentation và testing, guaranteed không route được trên internet. Dùng trong test để trigger lỗi kết nối mà không cần mock.
