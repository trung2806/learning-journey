## Thay đổi kỹ thuật

### 1. `@dataclass DaemonStats` — container metrics

```python
@dataclass
class DaemonStats:
    cycle_count: int = 0
    orphaned_cancel_count: int = 0
    sighup_count: int = 0
    timeout_node_count: int = 0
    alert_count: int = 0

STATS = DaemonStats()
```

**Tại sao `@dataclass` thay vì `dict`?**
- Attribute access `STATS.cycle_count` thay vì `STATS["cycle_count"]` — IDE autocomplete, typo được phát hiện ở compile time (AttributeError) thay vì runtime (KeyError)
- `@dataclass` tự generate `__repr__`, `__eq__` — debug và test dễ hơn
- `asdict(STATS)` từ `dataclasses.asdict` → dict → `json.dumps` trong một bước, không cần viết tay conversion

**Tại sao module-level singleton thay vì inject vào `main_loop`?**
- SIGUSR1 handler là `lambda` — không có tham số. Nếu STATS là local variable của `main_loop`, lambda phải capture nó qua closure (phức tạp và brittle hơn). Module-level singleton đơn giản và đủ — asyncio đơn luồng, không có race condition.
- Trade-off: module-level state khó test hơn (phải reset giữa các test). Giải quyết bằng `autouse=True` fixture trong `test_stats.py`.

### 2. SIGUSR1 handler

```python
loop.add_signal_handler(
    signal.SIGUSR1,
    lambda: logging.info(f"📊 [STATS DUMP] {json.dumps(asdict(STATS))}")
)
```

**Signal taxonomy sau D44:**
- `SIGTERM/SIGINT`: stop daemon gracefully
- `SIGHUP`: hot-reload fleet + thresholds
- `SIGUSR1`: dump stats to log on demand (inspect without stopping)

`loop.add_signal_handler` schedules callback trên event loop thread — không phải signal handler context thật (async-signal-unsafe). `logging.info` và `json.dumps` đều an toàn vì chạy trong event loop thread.

**Cách dùng runtime:** `kill -USR1 <pid>` → line log dạng:
```
2026-06-29 [INFO] 📊 [STATS DUMP] {"cycle_count": 142, "orphaned_cancel_count": 0, "sighup_count": 2, "timeout_node_count": 3, "alert_count": 7}
```

### 3. Instrumentation points (cả 5 fields)

| Field | Nơi tăng | Logic |
|---|---|---|
| `cycle_count` | Đầu outer while loop, sau SIGHUP check | Đếm mỗi cycle quét |
| `sighup_count` | Sau `_reload_event.clear()` | Đếm trigger, không phải success |
| `timeout_node_count` | Trong `_wrapped_poll` khi `asyncio.TimeoutError` | Đếm per-node, không per-cycle |
| `orphaned_cancel_count` | `finally` block trước khi `t.cancel()` | Đếm tasks bị kill sớm |
| `alert_count` | Sau `tick()` khi `alerts` không rỗng | `+= len(alerts)` — tích lũy |

**Nguyên tắc đặt instrumentation:** tất cả trong **imperative shell**, không bao giờ trong `tick()` hay `emit_alert()`. `tick()` là pure function — không side effect. Observability là side effect → thuộc imperative shell.

## Tests mới (`test_stats.py`)

**Pattern key:** `autouse=True` fixture `reset_stats` reset module-level `STATS` trước mỗi test — ngăn inter-test pollution. Kiểm tra bằng delta (`before + 1`) thay vì giá trị tuyệt đối — order-independent.

| Test | Loại | Mục đích |
|---|---|---|
| `test_daemon_stats_defaults` | Unit | Verify default values = 0 |
| `test_daemon_stats_asdict_json_serializable` | Unit | Verify `asdict()` → JSON serializable |
| `test_wrapped_poll_timeout_increments_timeout_count` | Integration | Delta check: +1 khi timeout |
| `test_wrapped_poll_success_does_not_increment_timeout_count` | Integration | Negative: không tăng khi success |
| `test_main_loop_cycle_count_increments` | Integration | cycle_count >= 1 sau >=1 cycle |
| `test_main_loop_sighup_increments_sighup_count` | Integration | sighup_count >= 1 sau SIGHUP |
| `test_sigusr1_does_not_stop_daemon` | Integration | Daemon tiếp tục sau SIGUSR1 |

## Ruling

- **Observability là side effect — thuộc imperative shell.** Functional core (tick, evaluate_metrics, _apply_node_reload) phải giữ pure. `STATS.xxx += 1` chỉ ở `main_loop` và `_wrapped_poll` (layer vận hành), không bao giờ trong tick().
- **Module-level singleton đủ tốt cho asyncio single-threaded context.** Race condition chỉ xảy ra với multi-threading. asyncio event loop là single-threaded → `STATS.field += 1` không cần lock. Nếu sau này expose qua HTTP API (multi-threaded server), cần chuyển sang `threading.Lock` hoặc atomic counter.
- **`@dataclass` cho metrics container + `asdict()` cho serialization** là idiom chuẩn cho in-process stats. Không cần thư viện ngoài (Prometheus/statsd) ở quy mô này.
- **`autouse=True` fixture để reset module-level state** là pattern bắt buộc khi test code có global singleton. Không reset → test order-dependent → flaky khi chạy song song hoặc reorder.
- **Delta assertion (`before + 1`) ổn định hơn absolute assertion** cho tests chạy sau tests khác có thể đã modify state.

## Glossary mới

- **Internal Metrics (In-Process Telemetry)**: counters/gauges lưu trong memory của process, không gửi ra external system. Zero latency, zero dependency. Giới hạn: mất khi restart, không aggregate nhiều process. Đủ cho single-daemon monitoring.
- **SIGUSR1 (User Signal 1)**: signal Unix dành riêng cho application-defined behavior. Convention phổ biến: dump state/stats. Không có semantic mặc định (kernel không làm gì với nó ngoài gửi tới process). Pair với SIGUSR2 cho các hành vi khác.
- **Signal Taxonomy cho daemon**: SIGTERM/SIGINT = stop (graceful shutdown), SIGHUP = reload config (zero downtime), SIGUSR1 = inspect state (no disruption). Ba signal, ba mục đích hoàn toàn khác nhau — không dùng lẫn.
- **`autouse=True` pytest fixture**: fixture tự động áp dụng cho mọi test trong scope (file hoặc conftest) mà không cần khai báo tham số. Dùng cho reset state, setup/teardown không cần assert.

## Open items

- `orphaned_cancel_count` đếm *số tasks bị cancel*, không đếm *số chu kỳ có orphaned task*. Với fleet 100 nodes tất cả timeout trong 1 chu kỳ, count = 100 dù chỉ 1 sự kiện. Có thể thêm `orphaned_cycles_count` nếu cần phân biệt.
- STATS reset về 0 khi daemon restart. Không persist qua restart. Nếu cần durability: flush ra file JSON khi nhận SIGTERM, load lại khi startup.
