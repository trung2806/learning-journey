## Quyết định: 1 exception + reason (không hierarchy)
Trục quyết định: type của exception là *contract để caller dispatch*. Ba reason map ra ba remediation thật:

| reason | nguyên nhân khả dĩ | Operator làm gì |
|---|---|---|
| `empty` | transport/timeout, device trả rỗng | retry / check reachability |
| `malformed` | non-JSON (lệnh sai / platform không hỗ trợ `\| as json`) | sửa platform dispatch, KHÔNG retry |
| `schema` | JSON hợp lệ sai cấu trúc (NOS đổi schema sau upgrade) | update parser |

Hierarchy 3 lớp *về nguyên tắc* đúng, nhưng người quyết granularity là caller, không phải engine. Chưa caller nào branch → giữ 1 type + attribute. Khi `build_row` (Day 16) hoặc log cần branch theo loại lỗi mới promote.

## srl_engine.py (final, đã verify)
```python
import json
from typing import Literal

ParseReason = Literal["empty", "malformed", "schema"]


class VersionParseError(Exception):
    """Lỗi tập trung cho mọi thất bại khi trích version.

    Mang `reason` (closed set) để Operator định hướng xử lý theo từng bề mặt lỗi,
    KHÔNG để caller phải đoán remediation từ message string.
    """

    def __init__(self, reason: ParseReason, detail: str):
        self.reason: ParseReason = reason  # "empty" | "malformed" | "schema"
        super().__init__(f"[{reason.upper()}] {detail}")


def parse_version(raw_json: str) -> str:
    """Engine thuần: raw JSON (Nokia SRL `show version | as json`) -> Software Version."""
    # 1. empty -> retry / check reachability
    if not raw_json or not raw_json.strip():
        raise VersionParseError(
            "empty",
            "Thiết bị phản hồi chuỗi rỗng hoặc mất kết nối trong lúc truyền dữ liệu.",
        )

    # 2. malformed -> sửa dispatch (lệnh sai / platform khác), KHÔNG retry
    try:
        data = json.loads(raw_json)
    except json.JSONDecodeError as e:
        raise VersionParseError(
            "malformed",
            f"Dữ liệu trả về không đúng định dạng JSON. Chi tiết: {e}",
        ) from e

    # 3a. schema -> precondition tường minh cho root (thay cho bắt TypeError rộng)
    if not isinstance(data, dict):
        raise VersionParseError(
            "schema",
            f"Root JSON là {type(data).__name__}, không phải object.",
        )

    # 3b. schema -> đường dẫn version. Access là MỘT biểu thức đơn nên KeyError/TypeError
    # ở đây CHỈ có thể là sai cấu trúc. Giữ block đúng 1 dòng để catch tuple không bao
    # giờ che được bug logic thêm vào sau.
    try:
        return data["basic system info"]["Software Version"]
    except (KeyError, TypeError) as e:
        raise VersionParseError(
            "schema",
            f"Cấu trúc JSON không khớp đường dẫn version: {e}",
        ) from e
```

## Review áp vào — và cái bẫy của chính refinement
Ba điểm review Day 15, áp 2/3 vào code:

1. **`isinstance` root guard thay `except TypeError` rộng.** Critique gốc ("broad TypeError che bug thêm vào sau") chỉ áp **yếu** ở đây vì try block là một biểu thức đơn — không có chỗ cho TypeError non-schema chui vào. Giá trị thật của refinement là: (a) precondition root tường minh, message rõ hơn cho ca phổ biến `null`/array; (b) document được vì sao catch còn lại an toàn (block 1 dòng).
2. **Bẫy đã suýt dính:** áp *ngây thơ* (chỉ xoá `except TypeError`) sẽ **regress coverage** — ca nhánh con không-dict (vd `{"basic system info": "oops"}`) raise TypeError ở subscript tầng 2, không được isinstance-root bắt, sẽ thoát ra dạng TypeError trần. Thuốc giải: giữ `except (KeyError, TypeError)` cho access (vẫn an toàn vì 1 dòng), và **viết test nested-not-dict** để chứng minh nhánh TypeError của catch tuple không thành dead coverage. Đây đúng lclass lesson "false coverage" — một refinement có thể lặng lẽ rớt coverage, test ca nested là cái pin lại.
3. **`reason: Literal[...]`** thay `str`: typo (`"shema"`) thành lỗi tooling (mypy/pyright) thay vì miss âm thầm khi caller branch. Lưu ý: Literal là static-only, KHÔNG enforce runtime — vẫn là cải thiện contract, không phải runtime guard.

## test_srl_engine.py (6 test, đã verify xanh)
Hai thay đổi so bộ Day 14: `test_..._invalid_root_type` giờ đi qua isinstance guard nên assert `__cause__ is None` (raise thẳng, không chain); **thêm** `test_..._nested_not_dict` để giữ coverage nhánh TypeError.
```python
import json

import pytest

from srl_engine import parse_version, VersionParseError

RAW_JSON_SAMPLE = """{
  "basic system info": {
    "Hostname": "srl1",
    "OS": "SR Linux",
    "Software Version": "v26.3.2",
    "Architecture": "aarch64"
  }
}"""


def test_parse_version_success():
    assert parse_version(RAW_JSON_SAMPLE) == "v26.3.2"


def test_parse_version_empty_or_whitespace():
    with pytest.raises(VersionParseError) as exc_info:
        parse_version("   \n   ")
    assert exc_info.value.reason == "empty"
    assert exc_info.value.__cause__ is None        # guard raise thang, khong chain


def test_parse_version_malformed_json():
    with pytest.raises(VersionParseError) as exc_info:
        parse_version("Error: unknown command or macro")
    assert exc_info.value.reason == "malformed"
    assert isinstance(exc_info.value.__cause__, json.JSONDecodeError)


def test_parse_version_schema_missing_key():
    with pytest.raises(VersionParseError) as exc_info:
        parse_version('{"basic system info": {"Hostname": "srl1"}}')
    assert exc_info.value.reason == "schema"
    assert isinstance(exc_info.value.__cause__, KeyError)


def test_parse_version_schema_invalid_root_type():
    # Root la array -> chan o isinstance guard, raise thang (khong chain)
    with pytest.raises(VersionParseError) as exc_info:
        parse_version('[{"Software Version": "v26.3.2"}]')
    assert exc_info.value.reason == "schema"
    assert exc_info.value.__cause__ is None


def test_parse_version_schema_nested_not_dict():
    # Nhanh con "basic system info" la string -> TypeError o subscript -> schema (co chain).
    # Test nay GIU cho nhanh TypeError cua catch tuple khong thanh dead/untested coverage.
    with pytest.raises(VersionParseError) as exc_info:
        parse_version('{"basic system info": "oops"}')
    assert exc_info.value.reason == "schema"
    assert isinstance(exc_info.value.__cause__, TypeError)
```

## Bài học hôm nay
- **Exception type là dispatch contract, granularity do caller quyết.** Không decide số exception trong trừu tượng; 1 type + `reason` attribute là điểm khởi đúng khi chưa ai branch, promote khi có consumer thật.
- **`from e` phải test bằng `__cause__`, không chỉ `pytest.raises`.** Test "ném đúng loại VÀ giữ đúng root cause" mới pin được chain; chỉ assert "có raise" là pin nửa vời.
- **Refinement có thể lặng lẽ rớt coverage.** Xoá `except TypeError` mà không xét nhánh con là regress; test nested-not-dict là cái chứng minh catch tuple còn sống. Một thay đổi "cho sạch" vẫn phải kèm test giữ invariant.
- **Broad-catch risk là về SIZE của block, không phải bản thân catch.** `except (KeyError, TypeError)` quanh một biểu thức đơn an toàn; rủi ro chỉ sinh khi nhồi thêm logic vào block. Comment hoá ràng buộc "giữ 1 dòng" cho người sau (có thể là mình).
- **Đóng session đúng ranh giới ≠ defer.** Backlog việc-đã-xong = 0 lúc đóng; cái mang sang là việc-chưa-bắt-đầu. Phân biệt này giữ forcing function khỏi bị lẫn với mechanism failure.

## Glossary thêm
- **Exception `reason` attribute (vs subclass hierarchy)**: gắn closed-set `reason` lên một exception thay vì dựng nhiều subclass. Caller catch base + branch theo `reason` khi cần; rẻ hơn hierarchy khi chưa có consumer branch, promote sau không phá call site.
- **`raise ... from e` + `__cause__`**: chain explicit giữ root cause; test pin bằng `assert isinstance(exc.__cause__, OriginalError)`. Raise thẳng (không `from`) → `__cause__ is None`, cũng nên assert để phân biệt nhánh guard vs nhánh chain.
- **`isinstance` precondition guard vs broad `except`**: validate kiểu tường minh ở đầu biến cái catch tình cờ thành điều kiện có chủ đích; mọi lỗi cùng loại thoát ra sau đó là bug thật, không bị dán nhãn nhầm. Đánh đổi: mất chain ở ca guard (raise thẳng).
- **`typing.Literal[...]` cho closed set**: ràng buộc giá trị hợp lệ ở tầng tooling (static), biến typo thành lỗi mypy/pyright. KHÔNG enforce runtime.
- **Broad-catch safety = single-expression block**: `except (KeyError, TypeError)` chỉ an toàn khi try body là một biểu thức; thêm logic vào body mở cửa cho exception non-mục-tiêu bị nuốt.
