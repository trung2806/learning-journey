### Việc 2 — Tách `parse_control_output` thành Pure Function + dùng output SR Linux thật trong `poll_node`

**Vấn đề:** `poll_node` từ D38 chạy lệnh `"show version"` (sai lệnh) và trả về hardcoded mock dict. `srl_fetcher.py` đã có parsing logic trong `SRLCliFetcher.fetch_control()` nhưng gắn chặt vào class netmiko — không share được với `poll_node` (asyncssh). Kết quả: logic parse bị nhân đôi hoặc drift.

**Input thật:** `control_A.json` — capture full-fidelity từ `"info from state platform control A | as json"`. Cấu trúc khớp hoàn toàn với mock D38: `cpu[].total.average-1`, `memory.utilization`, `temperature.instant/alarm-status`, `healthz.status`. Mock vốn đã đúng cấu trúc, chỉ cần nối vào data thật.

**Chuỗi quyết định:**

1. **Tách `parse_control_output` ra khỏi class.** Parsing logic = pure function, không cần `self`, không cần transport. Đặt ở cấp module trong `srl_fetcher.py` — cả `SRLCliFetcher` (netmiko) và `poll_node` (asyncssh) dùng chung mà không couple vào nhau.

2. **`CONTROL_CMD` là module-level constant.** Single source of truth cho chuỗi lệnh. Cả hai transport path dùng chung — `SRLCliFetcher.CONTROL_CMD` giờ là alias backward-compat, không phải definition.

3. **Exception taxonomy: transport vs data.** `SRLFetchError` = transport failure (SSH timeout, connection drop) — chỉ raise từ `fetch_control()` khi `send_command()` lỗi. `parse_control_output` raise stdlib exceptions: `ValueError` (rỗng/list rỗng), `json.JSONDecodeError` (non-JSON), `TypeError` (unwrap xong không phải dict). Lý do: pure function không nên phụ thuộc vào domain-specific error class; caller differentiate transport failure vs data failure.

4. **`fetch_control()` wrap network error, bubble parse error.** `try: raw = self.session.send_command(...)` → `except Exception → SRLFetchError`. Sau đó gọi `parse_control_output(raw)` mà không bọc — lỗi parse bubble lên tự nhiên. Caller cũ đang catch `SRLFetchError` cho transport; lỗi parse giờ bubble như stdlib exception.

5. **`poll_node` trong main.py:** `from srl_fetcher import CONTROL_CMD, parse_control_output`. Đổi lệnh SSH từ `"show version"` → `CONTROL_CMD`. Sau `conn.run()`: `return parse_control_output(result.stdout)`. Không cần import `SRLCliFetcher` — zero coupling giữa async transport và sync transport class.

6. **`parse_control_output` bắt lỗi list envelope:** SR Linux `| as json` đôi khi bọc object trong `[{...}]`. Unwrap phòng thủ: `if isinstance(data, list): data = data[0]`. Validated bằng `test_pipeline_unwraps_single_element_list_envelope` với real fixture.

## Ruling

- **Pure function isolation giải quyết transport coupling.** Khi hai transport layer (sync netmiko, async asyncssh) cần cùng parsing logic: tách logic thành top-level pure function, cả hai import trực tiếp. Không cần shared base class, không cần inheritance, không cần monkey-patch.
- **Exception taxonomy: transport vs data.** Domain exception (`SRLFetchError`) cho transport failure (network, authentication, timeout). Stdlib exceptions cho data failure (empty, bad JSON, wrong type). `safe_poll_node` catch `except Exception` bắt được cả hai — không cần thay đổi error boundary.
- **`CONTROL_CMD` là single source of truth cho command string.** Hai nơi dùng cùng lệnh mà không copy-paste = không bao giờ drift. Test `test_fetcher_sends_canonical_control_command` và `test_fetch_control_sends_correct_command` verify cả hai transport path dùng đúng constant này.
- **`FakeNetmiko` seam rõ ràng hơn `MagicMock` trực tiếp.** `FakeNetmiko` có explicit contract: `.send_command(cmd) -> str`, ghi `last_command`. `MagicMock` cho phép gọi bất kỳ method nào mà không lỗi — che giấu contract mismatch khi interface thật thay đổi. Seam tường minh là tài liệu sống của API.
- **Unwrap list envelope phải test với real fixture.** `"| as json"` đôi khi bọc trong list — test với fake string không phát hiện được nếu device thật có behavior khác. `test_pipeline_unwraps_single_element_list_envelope` dùng `json.dumps([json.loads(REAL_BYTES)])` để tạo wrapped version từ real bytes.

## Glossary mới

- **Test seam**: điểm trong code có thể "cắm" implementation thay thế không sửa production code. `SRLCliFetcher.__init__(session)` nhận object bất kỳ có `.send_command()` → test cắm `FakeNetmiko`, production cắm netmiko connection thật. Seam tốt = interface hẹp và rõ ràng.
- **Exception taxonomy**: hệ thống phân loại exception theo vai trò trong domain. Transport failure ≠ data failure ≠ programming bug — dùng khác type cho phép caller xử lý khác nhau mà không cần parse message string.
- **Provenance (xuất xứ dữ liệu)**: test `test_fetcher_sends_canonical_control_command` assert `ssh.last_command == "info from state platform control A | as json"` — verify rằng command đã sinh ra `control_A.json` và command đang được gửi lên device là một. Provenance khớp = fixture là ground truth thật.
