## Bài học hôm nay

1. **Single attachment point triệt hai FM bằng cấu trúc.** Khi `%` gắn ở đúng một hàm, FM4 (quên) cộng FM5 (đôi) không còn là hai lỗi rời mà quy về một câu "có đi qua `render_pct` không". FM4 bắt bằng pin human-output là `"6%"` (int trần làm test fail); FM5 chặn tại type vì `render_pct` nuốt int và reject str, không có chỗ thứ hai chèn `%`. Đây là cách biến một CLASS lỗi thành bất khả thay vì rình bắt từng case.
2. **Display layer không re-assert invariant của upstream.** Engine đã guard 0-100; `render_pct` re-check range sẽ tạo HAI chỗ khẳng định cùng một invariant, drift thì chỏi nhau, không rõ chỗ nào authoritative. `render_pct` chỉ guard cái thuộc về nó: TYPE (chặn double-application). Range thuộc engine. Cùng trục "đừng re-derive cái upstream đã đảm bảo" với không-cộng-components-thành-total.
3. **`%` không thuộc layer cả người lẫn máy đọc.** Nếu có sink máy, `%` nhét vào layer dùng chung là sai contract Prometheus (value phải unitless). `%` là mối quan tâm human-display thuần, sống ở renderer người; int engine chảy raw tới máy. Mở rộng đúng trục "không bake upstream" với threshold-thuộc-alert và %-không-thuộc-engine: giờ là %-không-thuộc-layer-dùng-chung.
4. **bool là int subclass, fail-loud phải reject tường minh.** `isinstance(True, int)` là `True`, nên guard chỉ `not isinstance(value, int)` để lọt bool. `True`/`False` trôi vào làm percent là bug. Check `isinstance(value, bool)` TRƯỚC, cùng tinh thần fail-loud "mọi field sai shape phải nổ" áp lên cả type sneaky.
5. **(process) stand-in xanh không bằng real-bytes xanh.** Fixture build khớp đúng giá trị anchor (6/2/28) nên test provenance với nó có phần vòng (build fixture để khớp assert). 16 test synthetic độc lập fixture là tín hiệu thật; 3 test chạm fixture chỉ đóng khi chạy trên capture thật. Đã chạy xanh capture thật để lock. Phân biệt rõ "code chạy đúng" (đã chứng) với "byte thật parse được" (phải chạy mới biết).

## Glossary thêm

- **Single attachment point (điểm gắn duy nhất)**: gắn một transformation (như hậu tố `%`) ở đúng MỘT hàm để hai failure mode đối ngẫu (quên gắn / gắn đôi) quy về một câu "có route qua hàm đó không". Quên bắt bằng pin output; gắn đôi chặn bằng reject input đã-định-dạng tại type. Biến class lỗi thành bất khả thay vì bắt từng case.
- **Type-guard vs range-guard (display layer)**: hàm hiển thị guard cái thuộc về nó (kiểu input, để chặn double-application) chứ không re-check domain invariant (0-100) mà upstream đã sở hữu. Re-check là nhân bản invariant, drift thì hai chỗ chỏi nhau, mất authoritative source.
- **bool-is-int-subclass footgun**: `isinstance(True, int)` là `True` nên guard chỉ-check-int để lọt bool. Reject bool tường minh TRƯỚC khi check int khi bool là input vô nghĩa.
- **Stand-in fixture circularity**: fixture tự build để khớp assert làm test provenance có phần vòng; chỉ test synthetic độc lập fixture cộng một lần chạy trên capture thật mới đóng được mỏ neo.

### `srl_display.py`

```python
from typing import Any

from srl_engine import parse_cpu_aggregate, parse_memory_utilization


def render_pct(value: int) -> str:
    """Render integer percent thành chuỗi cho người: 6 -> '6%'.

    Display layer, và là nơi DUY NHẤT gắn hậu tố '%'. Vì việc gắn xảy ra ở đúng
    một hàm, chỉ còn hai failure mode: quên gắn (caller đi vòng qua hàm này) và
    gắn đôi (apply hai lần). Cả hai quy về câu hỏi "value có đi qua đây không".

    Reject str và bool tại type boundary. str nghĩa là '%' có thể đã có sẵn từ
    upstream nên nhận vào sẽ cho phép double-application; bool là subclass của int
    nhưng vô nghĩa khi làm percent. Khoảng 0-100 CỐ Ý không re-check ở đây: invariant
    đó thuộc engine parser, không được nhân bản xuống display layer, cùng trục
    "không re-assert guarantee của upstream" như không re-derive total từ components.
    """
    if isinstance(value, bool) or not isinstance(value, int):
        raise TypeError(
            f"render_pct expects a plain int percent, got {type(value).__name__}"
        )
    return f"{value}%"


def render_metrics_for_humans(raw_json: dict[str, Any]) -> dict[str, str]:
    """View metric cho người: parse qua engine, render mỗi giá trị thành 'N%'.

    Nơi duy nhất int engine biến thành percent string. Sink máy (Prometheus
    exporter, JSON API) phải tiêu thụ int engine trực tiếp, KHÔNG đi qua đây, để
    path số trần không dính format hiển thị.
    """
    cpu = parse_cpu_aggregate(raw_json)
    memory = parse_memory_utilization(raw_json)
    return {
        "cpu_instant": render_pct(cpu.instant),
        "cpu_average_1": render_pct(cpu.average_1),
        "memory": render_pct(memory),
    }
```
