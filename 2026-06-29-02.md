## Việc làm hôm nay

### Fix 1 — stop_task per-cycle leak

**Vấn đề từ D41 horizon:** `stop_task` được tạo bên trong `while tasks` → mỗi vòng tạo 1 task mới:

```python
# TRƯỚC (bug)
while tasks and not _stop_event.is_set():
    stop_task = asyncio.create_task(_stop_event.wait())  # mới mỗi vòng
    done, _ = await asyncio.wait(tasks + [stop_task], FIRST_COMPLETED)
    if stop_task in done:
        break
    if not stop_task.done():
        stop_task.cancel()  # cancel nhưng KHÔNG await
```

Trace 3 nodes [A, B, C]:
- Vòng 1: tạo stop_task_1, A done → cancel stop_task_1 (không await)
- Vòng 2: tạo stop_task_2, B done → cancel stop_task_2 (không await)
- Vòng 3: tạo stop_task_3, C done → cancel stop_task_3 (không await), tasks rỗng → exit
- `finally`: tasks rỗng → orphaned = [] → stop_task_3 KHÔNG được cleanup

Mỗi cycle để lại 1 stop_task cancelled-but-not-awaited. Resolve khi stop_event fire, nhưng tích lũy qua nhiều cycle.

**Fix:** tạo `stop_task` một lần ngoài vòng lặp, cleanup trong `finally`:

```python
# SAU (fix)
stop_task = asyncio.create_task(_stop_event.wait())  # một lần cho toàn bộ chu kỳ
try:
    while tasks and not _stop_event.is_set():
        done, _ = await asyncio.wait(tasks + [stop_task], return_when=asyncio.FIRST_COMPLETED)
        if stop_task in done:
            break
        for fut in done:
            if fut in tasks:
                tasks.remove(fut)
                host, raw_data = fut.result()
                # process...
finally:
    if not stop_task.done():
        stop_task.cancel()
        try:
            await stop_task
        except asyncio.CancelledError:
            pass
    orphaned = [t for t in tasks if not t.done()]
    for t in orphaned:
        t.cancel()
    if orphaned:
        await asyncio.gather(*orphaned, return_exceptions=True)
```

**Tại sao `try: await stop_task` / `except CancelledError: pass` thay vì `gather(return_exceptions=True)`?** Cả hai đều đúng. `try/await` explicit hơn: tường minh là ta biết `CancelledError` sẽ đến và đang handle nó. `gather(return_exceptions=True)` ẩn exception — tốt khi clean up nhiều tasks cùng lúc.

### Fix 2 — Reactive Sleep Phase

**Vấn đề từ D41 horizon:** `await asyncio.wait_for(_stop_event.wait(), timeout=interval_seconds)` chỉ thức dậy khi stop hoặc timeout. SIGHUP fire lúc sleep → phải chờ hết `interval_seconds` mới xử lý reload.

**Fix:** race ba tasks cùng lúc:

```python
sleep_task = asyncio.create_task(asyncio.sleep(interval_seconds))
stop_task_sleep = asyncio.create_task(_stop_event.wait())
reload_task = asyncio.create_task(_reload_event.wait())

done_sleep, pending_sleep = await asyncio.wait(
    [sleep_task, stop_task_sleep, reload_task],
    return_when=asyncio.FIRST_COMPLETED
)

for t in pending_sleep:
    t.cancel()
if pending_sleep:
    await asyncio.gather(*pending_sleep, return_exceptions=True)
```

`asyncio.wait` trả về ngay khi BẤT KỲ task nào hoàn thành. Pending tasks phải được cancel + await để không leak.

**Pattern quan trọng:** Cancel + await là bắt buộc cho tasks không cần nữa. Chỉ cancel mà không await → task pending trong event loop cho đến khi được GC. `asyncio.gather(*pending, return_exceptions=True)` là idiom chuẩn cho multi-task cleanup.

**Bonus từ user:** SIGHUP reload cũng nạp lại cả `current_thresholds` (không chỉ nodes). Đồng bộ toàn diện hơn.

## Ruling

- **`asyncio.wait(tasks, FIRST_COMPLETED)` là primitive chuẩn cho event racing.** Khác với `asyncio.wait_for` (deadline cho 1 task), `asyncio.wait` race nhiều tasks và trả về ngay khi 1 hoàn thành. Dùng khi cần phản hồi với sự kiện đầu tiên trong nhiều nguồn (network task, stop signal, reload signal, timer).
- **`stop_task` phải sống qua toàn bộ vòng lặp xử lý, không tái tạo mỗi iteration.** Tái tạo mỗi vòng = tích lũy cancelled-but-not-awaited tasks. Quy tắc: 1 event → 1 task; task đó được reuse và cleanup trong finally.
- **Pending tasks từ `asyncio.wait` PHẢI được cancel + await.** Không cancel → tiếp tục chạy nền, dùng tài nguyên. Cancel nhưng không await → task pending cho đến GC, có thể log warning. Cancel + `asyncio.gather(*pending, return_exceptions=True)` → cleanup sạch.
- **Reactive sleep = thay `asyncio.wait_for(event.wait(), timeout)` bằng `asyncio.wait({sleep, stop, reload}, FIRST_COMPLETED)`.** Pattern cũ chỉ thức dậy sớm khi stop. Pattern mới thức dậy sớm khi BẤT KỲ sự kiện nào.

## Glossary mới

- **Event Racing (`asyncio.wait` + `FIRST_COMPLETED`)**: pattern race nhiều awaitables, xử lý ngay khi cái đầu tiên hoàn thành. Khác `gather` (đợi tất cả) và `wait_for` (deadline cho 1). Dùng khi có N nguồn sự kiện cần phản hồi.
- **Reactive Sleep**: sleep phase thức dậy sớm khi stop/reload, thay vì luôn chờ hết `interval_seconds`. Triển khai bằng race `[sleep_task, stop_task, reload_task]`.
- **Cancel + Await idiom**: `t.cancel()` sau đó `await asyncio.gather(t, return_exceptions=True)` (hoặc `try: await t; except CancelledError: pass`). Cancel schedules CancelledError; await đảm bảo task thực sự kết thúc. Thiếu await = leak.
