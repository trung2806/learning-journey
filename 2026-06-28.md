### (1) Error boundary nhất quán — FC/IS giữ nguyên, hai lớp tách biệt

`poll_node` D38 vừa là Raw Fetcher vừa là Error Boundary (có `except (asyncssh.Error, OSError)` trả fallback). `main_loop` có thêm `except* Exception` trong TaskGroup — dead code hoàn toàn vì task không bao giờ raise sau khi poll_node đã swallow. Hai lớp xử lý lỗi mâu thuẫn nhau.

**Chuỗi quyết định:**

1. **Actor Pattern vs FC/IS.** Actor Pattern (safe_poll_node tự quản hoàn toàn: transport + tick + state mutation + emit) bị loại vì hai lý do: (a) state_registry phân mảnh vào các concurrent task — mất Fleet-wide observability, khó tính correlation lỗi liên node sau này; (b) tick() mất bản chất Pure Function khi bị kéo vào task async. FC/IS giữ nguyên: state_registry là Single Source of Truth trong Imperative Shell, tick() vẫn testable độc lập.

2. **poll_node → Raw Fetcher thuần.** Bỏ toàn bộ `except` block khỏi `poll_node`. Nhiệm vụ duy nhất: mở SSH, chạy lệnh, trả dict thô. Không swallow bất kỳ lỗi nào.

3. **safe_poll_node → Layer 1 Transport Boundary.** Hai nhánh `except` tách biệt mục đích: `except (asyncssh.Error, OSError)` → `logging.error` + fallback `healthz.status="unreachable"` (lỗi vận hành thông thường, không phải bug); `except Exception` → `logging.critical` + fallback `healthz.status="error"` (bug lập trình chưa lường trước). Log level khác nhau để monitoring team phân biệt operational noise vs software defect.

4. **Xóa `except*` khỏi TaskGroup.** TaskGroup giờ chỉ làm structured concurrency — không phải error handler. Dead code xóa hết.

5. **Giữ `try/except Exception` trong processing loop.** Guard riêng cho bug trong `tick()` / `process_all_cooldowns()` — Layer 2, nhiệm vụ khác Layer 1.

**Hai lớp error boundary tách biệt:**
- Layer 1 (`safe_poll_node`): transport — luôn trả dict, không bao giờ raise. Phân biệt operational error vs bug.
- Layer 2 (processing loop): logic — bug trong tick() không kéo sập node khác.

**Test mới:** `test_safe_poll_node_catches_unhandled_bug` — patch `poll_node` raise `RuntimeError` → verify fallback `healthz.status="error"`, `reason` chứa message lỗi. `test_poll_node_fallback_behavior` đã migrate từ `poll_node` sang `safe_poll_node` (test `192.0.2.1` TEST-NET-1 → `status="unreachable"`).

**Phát hiện thêm:** 62 → 59 test sau khi xóa 4 test trùng khỏi test_main.py (3 load_thresholds duplicate + 1 tick_pure_logic_normal). Xác minh test_load_thresholds.py (5 tests) và test_tick.py (2 tests) vẫn còn nguyên — 59 là baseline sạch hơn 62, không phải mất coverage.

### (2) load_nodes() Fail-Fast + nodes.json externalize

Hardcoded `nodes = ["192.168.1.1", "192.168.1.2", "192.168.1.3"]` trong `main_loop` — không thay đổi được không restart daemon.

**Validation 5 lớp (Fail-Fast, pattern y chang load_thresholds D37):**

1. File không tồn tại / sai JSON cú pháp → `FileNotFoundError` / `json.JSONDecodeError` (tự raise từ stdlib).
2. Root không phải `list` → `TypeError`.
3. List rỗng → `ValueError` (fleet không có node = vô nghĩa).
4. Phần tử không phải `str` → `TypeError`.
5. Chuỗi IP không hợp lệ → `ValueError` (dùng `ipaddress.ip_address()` từ stdlib — cover IPv4 + IPv6, không cần regex).
6. IP trùng lặp → `ValueError` (IP là key của `state_registry` — duplicate gây ghi đè trạng thái và tạo Task thừa trong fan-out).

`load_nodes()` và `load_thresholds()` gộp trong cùng Fail-Fast block — daemon không khởi động được nếu thiếu hoặc sai một trong hai config.

`main_loop` signature thêm `nodes_path: str = "nodes.json"`. Entrypoint parse `sys.argv[2]` cho nodes_path.

**`test_load_nodes.py` — 8 tests với `tmp_path` fixture:** happy path (IPv4 + IPv6 mix), file not found, invalid JSON, not-a-list, empty list, non-string element, invalid IP, duplicate IP.

Chọn **Hướng A (List[str])** — giữ `safe_poll_node(host: str)` signature nguyên, không đổi kiến trúc. Dict-based format `[{"host": ..., "port": ..., "username": ...}]` là Horizon item, chưa cần.

### (3) asyncio.as_completed() — Reactive streaming

**Vấn đề với TaskGroup D38:** TaskGroup chờ ALL nodes xong mới thoát `async with`. Node A xong ở t=0.1s vẫn phải chờ Node B đến t=2.8s mới được xử lý — alert bị delay 2.7s không cần thiết.

**Giải pháp:** `asyncio.as_completed()` — xử lý từng node ngay khi nó xong, không block theo node chậm nhất.

**Chuỗi quyết định:**

1. **`asyncio.as_completed()` là iterator thường, không phải async iterator.** Duyệt `for fut in asyncio.as_completed(tasks):` — `for` thường. `await fut` bên trong. `async for` là lỗi syntax sai.

2. **Mapping problem:** `as_completed()` yield future theo thứ tự hoàn thành — không biết future nào là của host nào. Cần **identity-carrying wrapper**: hàm trả về `(host, data)` tuple thay vì chỉ `data`.

3. **`_wrapped_poll` định nghĩa bên trong `main_loop`, ngoài `while`.** Tránh tạo function object mới mỗi chu kỳ. Capture `safe_poll_node` từ outer scope.

4. **`asyncio.create_task()` thay TaskGroup.** Mất structured concurrency — task không tự cancel khi loop bị interrupt. Acceptable trong lab vì `safe_poll_node` bounded ~6s (connect_timeout 3s + command timeout 3s). Production cần explicit cancel.

5. **Reactive Timestamping: `now = time.time()` bên trong loop as_completed.** Mỗi node nhận timestamp tại thời điểm nó thực sự được xử lý — cooldown không bị drift do node chậm nhất.

**Code thay đổi trong `main_loop`:**
```python
async def _wrapped_poll(host_str: str) -> Tuple[str, Dict[str, Any]]:
    data = await safe_poll_node(host_str)
    return host_str, data

# Bên trong while:
tasks = [asyncio.create_task(_wrapped_poll(host)) for host in nodes]
for fut in asyncio.as_completed(tasks):
    host, raw_data = await fut
    now = time.time()  # Reactive Timestamping
    try:
        alerts, next_state = tick(raw_data, state_registry[host], now, ...)
        state_registry[host] = next_state
        for alert in alerts:
            emit_alert(host, alert)
    except Exception as bug:
        logging.critical(f"💥 [LOGIC BUG] {host}: {bug}", exc_info=True)
```

## Ruling

- **FC/IS over Actor Pattern khi cần Fleet observability.** Actor (task tự quản state) phân mảnh `state_registry` — mất khả năng tính correlation lỗi liên node, aggregate báo cáo fleet. FC/IS giữ state tập trung, tick() pure → testable. Chọn Actor khi unit của xử lý là node hoàn toàn độc lập và không cần cross-node logic.
- **Hai nhánh `except` trong Transport Boundary phân biệt operational vs bug.** `asyncssh.Error` + `OSError` là lỗi vận hành bình thường (logging.error, unreachable). `Exception` còn lại là bug phần mềm (logging.critical, error). Dùng chung một nhánh `except Exception` cho cả hai = spam CRITICAL mỗi lần node offline.
- **`asyncio.as_completed()` là iterator thường.** `for fut in ...:` rồi `await fut` bên trong. Không phải `async for`. Không biết future nào của task nào → cần identity-carrying wrapper.
- **Identity-carrying data:** trả `(identifier, payload)` để context đi theo dữ liệu qua pipeline — thay vì caller track mapping ngoài.
- **Reactive Timestamping:** `now = time.time()` trong as_completed loop = mỗi node lấy timestamp lúc được xử lý thật, không bị drift bởi node chậm nhất.
- **`create_task()` + `as_completed()` đánh đổi structured concurrency lấy streaming.** TaskGroup: wait-for-all + auto-cancel; as_completed: process-as-ready + orphaned task khi interrupt. Lab chấp nhận vì transport bounded. Production cần explicit cancel.

## Glossary mới

- **`asyncio.as_completed(awaitables)`**: nhận iterable of awaitables/futures, trả về iterator thường. Yield từng future theo thứ tự HOÀN THÀNH (không phải thứ tự tạo). `await fut` bên trong `for` để lấy kết quả. Không biết future nào là của coroutine nào → cần wrapper mang identity.
- **Identity-carrying data**: pattern thiết kế pipeline — hàm trả về `(identifier, payload)` tuple thay vì chỉ payload. Context (host IP, request ID...) đi theo dữ liệu xuyên suốt, tránh caller phải tự maintain mapping ngoài.
- **Reactive Timestamping**: tính `now = time.time()` ngay tại điểm xử lý từng đơn vị (per-node, per-event) thay vì một lần trước fan-out. Đảm bảo timestamp phản ánh thời điểm quyết định thật, không bị lệch do latency của các đơn vị khác trong cùng batch.
- **Transport Boundary (Layer 1)**: lớp wrapper bao quanh I/O transport, đảm bảo không bao giờ raise ra ngoài — luôn trả về cấu trúc dữ liệu chuẩn (hoặc fallback). Phân biệt với Logic Boundary (Layer 2) guard lỗi tính toán.
