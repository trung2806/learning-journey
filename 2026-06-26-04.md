## Việc làm hôm nay: externalize threshold → config file JSON

`DEFAULT_THRESHOLDS` hardcode trong `main.py` — không thể thay đổi ngưỡng mà không sửa code. Hôm nay tách ra file `thresholds.json` và viết `load_thresholds()` với validation đầy đủ.

**Chuỗi quyết định (mentor dialogue, Trung tự chốt từng bước):**

1. **Config loading là I/O → Imperative Shell, không phải Functional Core.** Đọc file từ disk là side-effect, có thể fail theo trạng thái OS. `tick()` không bao giờ đọc file — nó nhận `thresholds` đã là dict thuần túy.
2. **Vị trí trong main_loop(): TRƯỚC while.** Hai lý do: (1) Hiệu năng — I/O một lần duy nhất khi khởi động, kết quả trên RAM cho mọi vòng lặp; (2) Fail-Fast — lỗi cú pháp file nổ ngay khi gõ lệnh chạy, không crash giữa production.
3. **Format JSON (stdlib), không YAML.** `import json` không sinh dependency bên thứ ba — quan trọng trong môi trường air-gapped và data center bảo mật cao nơi cài pip package ngoài cần quy trình phê duyệt.
4. **Fail-Fast (A), không silent fallback.** Nếu file mất mà daemon fallback im lặng về DEFAULT_THRESHOLDS, hệ thống chạy với ngưỡng sai so với production mà không ai hay — lọt cảnh báo nghiêm trọng.
5. **Validation ba lớp:** (1) type check root là dict; (2) required keys `{cpu, memory, temperature}` đủ; (3) từng value: `isinstance(int)` + guard `bool` + `> 0`.
6. **Bool guard trong isinstance check.** `isinstance(True, int)` trả về `True` vì `bool` là subclass của `int` trong Python. `{"memory": true}` trong JSON sẽ pass qua validator nếu thiếu guard. Fix: `not isinstance(val, int) or isinstance(val, bool)`.
7. **Return subset whitelist, không toàn bộ dict.** Key lạ (`latency`) chưa có validation logic → cho qua là break contract "dữ liệu sạch đã validate". Hàm trả về chỉ 3 key bắt buộc.
8. **Exit code: re-raise thay vì `sys.exit(1)` trong main_loop().** `sys.exit()` trong library code = "kẻ sát thủ" process, phá vỡ khả năng import. `main_loop()` log CRITICAL rồi `raise`; `if __name__ == "__main__":` bắt và gọi `sys.exit(1)`. Tách Cơ chế (mechanism) khỏi Chính sách (exit policy).
9. **Dependency Injection cho config_path.** `main_loop(config_path: str = "thresholds.json")` — không hardcode path trong hàm, không đọc `sys.argv` bên trong loop. `if __name__` đọc `sys.argv[1]` nếu có rồi truyền vào. Giúp test có thể inject path tùy ý qua `tmp_path`.

## Code mới: `load_thresholds()` + `test_load_thresholds.py` + `thresholds.json`

**`load_thresholds(filepath: str) -> Dict[str, int]`:**
- `open()` + `json.load()` — FileNotFoundError và json.JSONDecodeError tự propagate
- `isinstance(config, dict)` → TypeError nếu không phải dict
- `required_keys - config.keys()` → ValueError nếu thiếu key (set subtraction, Pythonic)
- Loop qua `required_keys`: `isinstance(val, bool)` guard trước, `isinstance(val, int)` sau, `val <= 0` cuối
- Build và return `validated_thresholds` (subset whitelist)

**`main_loop()` thay đổi:**
- Thêm tham số `config_path: str = "thresholds.json"`
- Gọi `load_thresholds()` trước `while`, bắt 4 exception type, log CRITICAL rồi `raise`
- `tick()` nhận `current_thresholds` (từ RAM) thay vì `DEFAULT_THRESHOLDS` (hardcode)
- `if __name__`: đọc `sys.argv[1]` nếu có, bọc `main_loop()` trong `try/except Exception: sys.exit(1)`

**`thresholds.json`:**
```json
{"cpu": 80, "memory": 25, "temperature": 75}
```

**`test_load_thresholds.py` — 5 test case, mỗi test dùng `tmp_path` fixture:**

- TypeError: root là list; value là string; value là bool → 3 sub-case một test

## Ruling

- **Config loading là I/O → Shell.** Mọi thứ đọc file, gọi OS clock, gọi network đều sống bên ngoài Functional Core. `tick()` nhận dict thuần túy, không biết file tồn tại hay không.
- **Bool là subclass của int — Python-specific gotcha.** Không phải lỗi ngữ nghĩa thông thường. Validation layer phải có guard `isinstance(val, bool)` trước `isinstance(val, int)` để loại `True`/`False`.
- **Whitelist return > passthrough.** Trả về subset đã validate = contract tường minh. Caller biết chính xác những gì nhận được. Key lạ không có logic validate thì không được qua.
- **Mechanism vs Policy — sys.exit() không thuộc về library.** `main_loop()` là mechanism — điều phối, log, raise. `if __name__` là policy — quyết định exit code cho OS. Trộn hai thứ = không thể test, không thể nhúng vào hệ thống lớn hơn.
- **`tmp_path` là fixture đúng cho I/O test.** Không cần mock, không cần `tempfile` thủ công — pytest tạo và dọn tự động, mỗi test có thư mục biệt lập.

## Glossary mới

- **`tmp_path` (pytest fixture).** Pytest built-in fixture trả về `pathlib.Path` trỏ vào thư mục tạm biệt lập per-test. Test hàm đọc/ghi file mà không cần mock I/O và không cần dọn dẹp thủ công. Inject vào test function như tham số: `def test_foo(tmp_path):`.
- **Dependency Injection qua tham số hàm.** Thay vì hardcode dependency (file path, URL, config object) bên trong hàm, truyền nó qua tham số. Hàm linh hoạt, testable, và nhúng được vào context khác mà không sửa body. Pattern xuất hiện nhất quán suốt Phase 1: `current_time`, `thresholds`, `config_path` đều inject qua parameter.
- **Whitelist return pattern.** Hàm validator chỉ trả về subset key đã được kiểm tra, bỏ qua key lạ. Contract tường minh: caller biết chính xác shape và tính hợp lệ của dữ liệu nhận được — không bị surprise bởi key chưa validate.
- **Set subtraction cho missing key check.** `required_keys - config.keys()` → set các key bị thiếu. Pythonic, O(n), không cần vòng lặp tường minh.
