## Quyết định 1: tách guard khỏi comparison (HAI helper, không gom một)

`_guard_threshold(threshold) -> int` và `_check_breach(value, threshold) -> bool` là hai helper độc lập, KHÔNG gộp một hàm.

Hai trục trách nhiệm khác nhau. `_guard_threshold` là Mép module (boundary defense): sở hữu type-contract, nổ `TypeError` lập tức nếu caller truyền sai dạng (reject `bool` int-subclass footgun, reject non-int). `_check_breach` sở hữu toán tử logic so sánh thuần (`>`). Lý do tách KHÔNG phải thẩm mỹ mà là một kịch bản cụ thể: một vòng lặp quét nhiều `value` share một `threshold` đã validate sẵn sẽ buộc guard lặp đi lặp lại thừa thãi nếu gom chung. Tách giữ mỗi helper tinh khiết, không chồng trách nhiệm. (Đây là quyết định thiết kế "hai-hay-một" riêng biệt với quyết định timing "khi nào trích"; timing đã đúng từ Day 23/24, chờ consumer thứ ba mới trích.)

## Quyết định 2: định tuyến dữ liệu, trích value SẴN trước khi gọi helper

`_check_breach(value, threshold)` chỉ nhận hai scalar `int` trần. Khâu chọn trường (field-selection) nằm lại tại từng evaluator: `evaluate_cpu_alert` đọc `agg.average_1` rồi mới gọi helper; memory/temperature truyền thẳng scalar đã có.

Lý do giữ engine/IO split sạch: khế ước cấu trúc dữ liệu thô của mỗi metric khác nhau (cpu mang container phức hợp chứa `average_1`; memory và temperature là scalar trần). Nếu đẩy field-selection vào helper dùng chung, helper buộc nhận Union `CpuAggregate | int` và phải `isinstance` rẽ nhánh, một anti-pattern đẩy nhận thức về data shape sai tầng, phá encapsulation đã xây cho `MetricAlert`. Helper ở dưới phải mù về shape; tầng trên bóc shape rồi mới gọi.

## Quyết định 3: `parse_temperature`, current-not-margin pin ở PARSER

`parse_temperature(raw_json) -> int` navigate `control[0]["temperature"]["sensor"]` (một list cảm biến giống cpu), chọn entry `index == "all"`, đọc `current`, loại bỏ `margin`. `margin` là khoảng-cách-tới-ngưỡng, semantic khác hẳn nhiệt độ tuyệt đối nên không bao giờ là field đúng để alert.

KHÔNG range-check 0-100: temperature KHÔNG phải phần trăm, âm hoặc trên 100 đều là giá trị vật lý hợp lệ. cpu/memory range-check vì chúng là %; đây là domain invariant theo-từng-metric, đặt ở parser, có cho metric %, KHÔNG cho temperature. (Quyết định-omission này hiện CHƯA pin, sang open item discharge-first.)

Field-choice pin ở tầng nào trôi theo tầng còn nhiều field để chọn. cpu mang `CpuAggregate(instant, average_1)` xuống alert nên field-choice (average không instant) pin Ở ALERT bằng straddle. temperature parser trả scalar trần nên alert không bao giờ thấy `margin`; field-choice (current không margin) BẮT BUỘC pin Ở PARSER bằng crosswire (bơm margin rất cao, current thấp, assert parser chỉ bốc current). Cùng nguyên lý quyết-định-đọc-field-phải-có-test-chỉ-đúng-mới-pass, khác tầng đặt test.

## Glossary thêm

- **Một-công-việc-trip-nhiều-gate (rule-of-three đa trục)**: một đơn vị new-work có thể là caller thứ ba của nhiều pattern đồng thời (temperature = thứ ba của `control[0]`-navigation VÀ guard+comparison). Đếm rule-of-three theo từng pattern độc lập; fire mọi gate đủ điều kiện trong cùng session; không để gate này che gate kia.
- **Guard tách Comparison (hai trục trách nhiệm)**: `_guard_threshold` sở hữu type-contract ở mép module; `_check_breach` sở hữu toán tử logic. Tách để loop-share-validated-threshold không guard lặp. Hai helper, không gom một. Trục riêng với "khi nào trích" (rule-of-three).
- **Field-choice pin theo tầng-mang-data (parser vs alert)**: nơi pin quyết-định-đọc-field trôi theo tầng mà data còn nhiều field để chọn. Container xuống alert sang pin ở alert (cpu average-not-instant, straddle). Scalar trần xuống alert sang pin ở parser (temperature current-not-margin, crosswire). Cùng họ với straddle/crosswire field-selection.
- **Invariant dương cho omission (pin cả việc KHÔNG làm)**: một quyết định bỏ check (temperature không range-check vì không phải %) cần test dương chứng nó được giữ (`current=130` parse sạch), nếu không một check thêm vào sau lọt im. Omission cần pin y như presence cần rejection. Đối xứng với accepts-out-of-range của alert layer.
- **Range-check là domain invariant theo-từng-metric**: check 0-100 thuộc về metric LÀ phần trăm (cpu, memory), đặt ở parser. Temperature không phải % nên không check. Invariant domain không nhân bản mù sang mọi metric; nó theo bản chất đại lượng.

### `srl_engine.py` (thêm helper dùng chung + parser temperature)

```python
def _extract_control_zero(raw_json: dict[str, Any]) -> dict[str, Any]:
    """Helper dùng chung trích control[0] (Rule-of-three nổ, parser thứ ba).
    Fail-loud tại cửa ngõ nạp dữ liệu cấu trúc.
    """
    if not isinstance(raw_json, dict):
        raise ValueError(f"expected dict payload, got {type(raw_json).__name__}")
    controls = raw_json.get("control")
    if not isinstance(controls, list) or not controls:
        raise ValueError("payload has no non-empty 'control' list")
    control = controls[0]
    if not isinstance(control, dict):
        raise ValueError("control[0] is not an object")
    return control


def parse_temperature(raw_json: dict[str, Any]) -> int:
    """Platform temperature celsius từ sensor index='all'. Bốc 'current',
    loại 'margin'. KHÔNG range-check (temperature không phải %).
    """
    control = _extract_control_zero(raw_json)
    temp_block = control.get("temperature")
    if not isinstance(temp_block, dict):
        raise ValueError("control[0] has no 'temperature' object")
    sensors = temp_block.get("sensor")
    if not isinstance(sensors, list) or not sensors:
        raise ValueError("temperature object has no non-empty 'sensor' list")
    target_sensor = next(
        (s for s in sensors if isinstance(s, dict) and s.get("index") == "all"),
        None,
    )
    if target_sensor is None:
        raise ValueError("no temperature sensor entry with index 'all'")
    current = target_sensor.get("current")
    if current is None:
        raise ValueError("temperature sensor entry is missing 'current'")
    try:
        current_int = int(current)
    except (ValueError, TypeError):
        raise ValueError(
            f"temperature 'current' must be an integer, got {type(current).__name__}"
        )
    return current_int
```

### `srl_alert.py` (trích helper + evaluator temperature)

```python
def _guard_threshold(threshold: int) -> int:
    """Type Boundary của policy ngoài tại cửa ngõ alert. Chặn bool + non-int.
    KHÔNG range-check.
    """
    if isinstance(threshold, bool) or not isinstance(threshold, int):
        raise TypeError(
            f"threshold expects a plain int, got {type(threshold).__name__}"
        )
    return threshold


def _check_breach(value: int, threshold: int) -> bool:
    """Toán tử so sánh lõi dùng chung (Rule-of-three nổ, consumer thứ ba)."""
    return value > threshold


def evaluate_temperature_alert(value: int, threshold: int) -> MetricAlert:
    validated_threshold = _guard_threshold(threshold)
    breached = _check_breach(value, validated_threshold)
    return MetricAlert(
        metric="temperature_celsius",
        value=value,
        threshold=validated_threshold,
        breached=breached,
    )
```

`evaluate_cpu_alert` và `evaluate_memory_alert` đổi sang gọi `_guard_threshold` + `_check_breach` thay cho guard/compare inline; `evaluate_cpu_alert` giữ field-selection `value = agg.average_1` tại chỗ.
