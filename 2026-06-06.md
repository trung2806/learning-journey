## port_check — fix A (quan trọng nhất)
Bug Day 11: regex `-\s*(\d+)` over-match. Trên allowlist có comment `# range 8000-8080` + dòng `owner: team-net-01`, regex nhét 8080 và 1 vào allowed_ports => cổng lạ 8080 KHÔNG bị báo (false-negative, fail-open).

Fix:
```python
import yaml  # bo import re
...
if allowlist_path.exists():
    data = yaml.safe_load(allowlist_path.read_text(encoding="utf-8"))
    allowed_ports = {int(p) for p in (data or {}).get("allowed_ports", [])}
```
- File thiếu => allowed_ports rỗng => mọi cổng bị báo = fail-closed, đúng hướng cho tool bảo mật (ngược regex).
- Type: int cả hai vế (yaml ra int, port từ ss ép int) => so sánh int-vs-int sạch.
- Điểm nhỏ chấp nhận: port không phải số trong yaml sẽ raise ValueError không bắt = fail-loud lúc load config, ok.
- **Bài học lặp lại:** lesson đã ghi Day 11 mà vẫn ship bản regex. Diagnosis ≠ application. Lần sau chốt fix là sửa NGAY trong cùng session, không để treo.

## Glossary thêm
- **yaml.safe_load vs regex cho config**: parse config có cấu trúc bằng parser đúng định dạng, KHÔNG bóc bằng regex. Regex over-match => allowlist tự nới => fail-open (hướng hỏng nguy hiểm nhất cho tool bảo mật).
- **MagicMock(stdout=...)**: mock một object có thuộc tính, khác Day 8 (mock hàm trả scalar). Dùng khi code đọc `result.stdout`.
- **textwrap.dedent**: triệt tiêu indent chung của block string, cho fixture test / heredoc không phụ thuộc thụt lề source.
- **ConnectHandler (netmiko)**: mở SSH session tới thiết bị, dùng làm context manager (`with`) để tự đóng. `device_type` chọn driver (vd `nokia_srl`, `cisco_ios`). `.send_command()` chạy lệnh, trả stdout đã cắt echo + prompt.
- **containerlab deploy -t <topo>.clab.yml**: dựng lab container từ file topology. Node có `kind` (nokia_srlinux, arista_ceos, linux...) và `type` (chassis variant).
