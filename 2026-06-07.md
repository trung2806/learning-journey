## Decision 2 — find_stale contract (khoá)
Spec đã chốt, dán làm docstring đầu file:
```python
"""find_stale.py — Python thay cho `find -type f -mtime +N`.

- Quét: mọi file (default glob "*"), đệ quy. KHÔNG narrow về *.log.
- Output: mỗi dòng gồm cột tuổi (ngày) + đường dẫn.
- Bien: age_seconds > days_threshold * 86400 (strict, theo giây).
  File đúng tuổi days_threshold bị LOẠI. Pin bằng test.
- Chữ ký: find_old_files(directory: Path, days_threshold: int)
"""
```
Cách làm (quan trọng hơn kết quả): **reconcile, không rewrite**. Gõ lại từ trí nhớ chính là cái đẻ ra drift lần 1, 2, 3. Lấy bản Day 10 thật từ git rồi reconcile có chủ đích:
```bash
git log --oneline -- find_stale.py     # tìm commit Day 10
git show <hash>:find_stale.py          # xem bản Day 10 gốc
git diff <hash> -- find_stale.py        # diff với hiện tại
```
Giữ default `*` + cột tuổi của Day 10; áp tên `days_threshold` của Day 11 (đổi có chủ đích, không phải drift); pin biên. Sau đó pin biên 30.0 ngày bằng một test (file đúng 30.0 ngày bị loại, vì dùng strict `>`). Hôm nay là dịp áp đúng lesson Day 12 NGAY trong session.

Note biên: chọn `>` strict-by-seconds (Day 10) vì đã hiểu rõ và sạch. Nếu đổi sang `>=` inclusive-by-day thì cũng được, miễn deliberate và pin test, KHÔNG để hai semantics cùng tồn tại ở hai file như Day 11.
