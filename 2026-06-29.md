### Item 1 — Cancel Orphaned Tasks

**Vấn đề:** `_stop_event.set()` fire giữa `for fut in as_completed(tasks)` không tự cancel các task chưa được `await`. Chúng trở thành orphaned coroutines — tiếp tục chạy ngầm, dùng tài nguyên, và Python log warning "Task destroyed but it is pending".

**Fix:** Wrap `for fut in as_completed(tasks)` trong `try/finally`. `finally` block luôn chạy dù break sớm hay exception:

```python
try:
    for fut in asyncio.as_completed(tasks):
        host, raw_data = await fut
        if _stop_event.is_set():
            break
        # ... process
finally:
    orphaned = [t for t in tasks if not t.done()]
    for t in orphaned:
        t.cancel()
    if orphaned:
        await asyncio.gather(*orphaned, return_exceptions=True)
```

`t.cancel()` inject `CancelledError` vào next `await` point của task. `asyncio.gather(..., return_exceptions=True)` bắt `CancelledError` như result (không re-raise) và return ngay khi tất cả cancel xong.

### Item 2 — Per-Node Timeout (Outer Watchdog)

**Vấn đề:** `asyncio.as_completed` không có deadline per-task. Node cực chậm (hung SSH, blocked socket) block `await fut` indefinitely — daemon không thoát được.

**Fix:** `_wrapped_poll` được move lên module level và thêm `asyncio.wait_for`:

```python
POLL_TIMEOUT_SECONDS = 10  # outer watchdog

async def _wrapped_poll(host_str: str, timeout: float = POLL_TIMEOUT_SECONDS) -> Tuple[str, Dict[str, Any]]:
    try:
        data = await asyncio.wait_for(safe_poll_node(host_str), timeout=timeout)
    except asyncio.TimeoutError:
        data = {"healthz": {"status": "timeout", "reason": f"poll exceeded {timeout}s"}, ...}
    return host_str, data
```

**Tại sao `POLL_TIMEOUT_SECONDS = 10` không phải 6?**

Hai lớp ranh giới thời gian cần tách biệt rõ:
- **Inner timeouts** (asyncssh layer): `connect_timeout=3` + SSH command `timeout=3` = 6s worst-case. Cho phép asyncssh tự xử lý lỗi mạng và raise exception chính xác.
- **Outer watchdog** (lifecycle layer): `POLL_TIMEOUT_SECONDS = 10`. Buffer 4s đảm bảo 2 điều:
  1. **Bảo toàn error taxonomy**: asyncssh.Error/OSError nổ trước → `safe_poll_node` phân loại đúng. Nếu outer == inner: race condition khiến `asyncio.TimeoutError` cướp trước, mất thông tin lỗi chi tiết từ tầng mạng.
  2. **Resource cleanup**: asyncssh có thời gian đóng socket sạch (TCP FIN/RST) trước khi bị kill từ bên ngoài. Tránh zombie sockets, fd leaks.

**Quy tắc bất biến:** Outer Timeout > tổng Inner Timeouts. Khoảng đệm tối thiểu ~30-50%.

### Item 4 — SIGHUP Fleet Hot-Reload

**Vấn đề:** Thêm/bớt node vào `nodes.json` yêu cầu restart daemon. Với SIGHUP, có thể reload danh sách mà không downtime.

**Hai thành phần:**

**A. `_apply_node_reload` — pure function:**

```python
def _apply_node_reload(new_nodes, current_registry) -> Dict[str, SystemAlertState]:
    return {h: current_registry.get(h, SystemAlertState()) for h in new_nodes}
```

Logic merge: node còn trong list → giữ nguyên `SystemAlertState` cũ (không reset cooldown); node mới → `SystemAlertState()` fresh; node bị xóa → drop. Pure function, không side effect, testable trực tiếp.

**B. Tích hợp vào `main_loop`:**

```python
_reload_event = asyncio.Event()
loop.add_signal_handler(signal.SIGHUP, lambda: _reload_event.set())

# Đầu mỗi chu kỳ:
if _reload_event.is_set():
    _reload_event.clear()
    new_nodes = load_nodes(nodes_path)
    state_registry = _apply_node_reload(new_nodes, state_registry)
    nodes = new_nodes
```

Reload check ở đầu mỗi chu kỳ, không interrupt sleep. Hiệu lực trong vòng `interval_seconds` sau khi nhận tín hiệu — acceptable cho daemon.

## Ruling

- **Move `_wrapped_poll` lên module level là prerequisite cho testability.** Closure nằm trong `main_loop` không thể test trực tiếp qua `main._wrapped_poll`. Rule: bất kỳ logic có nhiều hơn 1 code path đều cần được test — nếu không test được, refactor ra module level.
- **`finally` là nơi đúng để cancel orphaned tasks, không phải `except`.** `except` chỉ chạy khi có exception. `finally` chạy khi `break` sớm, khi exception, khi return — mọi exit path.
- **`asyncio.gather(*tasks, return_exceptions=True)` là idiom chuẩn để await cancelled tasks.** `return_exceptions=True` bắt `CancelledError` như return value, không re-raise. Gọi `gather` sau `cancel` để đảm bảo cleanup hoàn tất trước khi exit.
- **SIGHUP signal handler chỉ set event, không làm logic nặng.** Async-Signal Safe: signal handler chạy trong context bất kỳ, không có asyncio event loop. `asyncio.Event.set()` là thread-safe. Logic reload chạy trong coroutine context (đầu while loop), không trong signal handler.
- **`_apply_node_reload` pure function cho phép test state merge mà không cần chạy main_loop.** Tách pure logic ra khỏi imperative shell = test rẻ và nhanh.

## Glossary mới

- **Outer Watchdog Timeout**: timeout bọc ngoài toàn bộ chuỗi IO, để bảo vệ vòng đời task. Phân biệt với inner timeouts (của từng thư viện). Quy tắc: outer > tổng inner; buffer ≥ 30-50%.
- **Orphaned Task**: asyncio.Task được tạo nhưng không được `await` khi event loop thoát hoặc parent coroutine exit sớm. Biểu hiện: `Task destroyed but it is pending` trong log. Fix: `finally` + `cancel()` + `gather(return_exceptions=True)`.
- **`asyncio.Event` vs `asyncio.Event.set()` in signal handler**: `Event.set()` là thread-safe, có thể gọi từ signal handler. Trái lại, các asyncio primitives khác (Queue, Lock) không thread-safe khi gọi ngoài coroutine context.
- **Hot-reload via SIGHUP**: pattern daemon Unix truyền thống — SIGHUP = "reload config, không restart". Trong asyncio: signal handler set event, coroutine check event ở điểm an toàn (đầu chu kỳ), xử lý reload trong coroutine context.
