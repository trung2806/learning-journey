Test trên allowlist có comment "range 8000-8080" + `owner: team-net-01`: regex ra `{'22','80','443','01','8080'}`, tự nhét 8080 và 01 vào allowlist. yaml.safe_load ra sạch `{22,80,443}`. Với tool allowlist cổng, parser tự nới allowlist là false negative: cổng lạ bị coi là hợp lệ. Đã dùng yaml.safe_load ở Day 9, đây là bước lùi.
  - Kèm theo: yaml.safe_load trả INT, port từ ss là STRING. Phải chuẩn hoá cùng kiểu (ép int(port) ở vế ss), không thì `"22" not in {22}` luôn True, mọi cổng bị báo lạ.

- **C (nhất quán + testability): sys.exit đặt sai chỗ.** load_allowlist gọi sys.exit bên trong (trộn load với control flow, giết process khi test). main() cũng sys.exit bên trong thay vì pattern Day 5/6/10: `def main() -> int: return code` rồi `sys.exit(main())`. ## Glossary thêm (3 mục)
- **tmp_path**: fixture pytest cấp Path thư mục tạm riêng mỗi test, tự dọn. Khai làm tham số hàm test là có.
- **os.utime(path, (atime, mtime))**: set lại access/modify time, epoch giây. Dùng "lùi tuổi" file trong test.
- **subprocess.run(cmd, capture_output=True, text=True, check=True)**: chạy lệnh ngoài, lấy stdout dạng str (text=True), raise CalledProcessError nếu exit != 0 (check=True). Khác Day 5 (chỉ xem returncode) ở chỗ giờ lấy và parse output.
